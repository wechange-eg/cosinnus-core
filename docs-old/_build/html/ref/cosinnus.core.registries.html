cosinnus.core.registries.base module¶

The cosinnus app ecosystem allows apps to provide certain functionality that can be used from cosinnus core as well as other (cosinnus) apps.

The initial implementation adapts the ActionRegistry or DynamicFormFieldRegistry from django-dynamic-forms https://github.com/Markush2010/django-dynamic-forms/tree/f306619e12

class cosinnus.core.registries.base.BaseRegistry[source]¶

Bases: object

The BaseRegistry provides a basic layout each registry should use. The main idea is to provide locking while registering new objects.

Subclasses must implement a register() function.

register(*args, **kwargs)[source]¶

Registers a new item in the registry.

Implement something like:

def register(self, *args, **kwargs):
    with self.lock:
        # Here comes my fancy registration logic
class cosinnus.core.registries.base.DictBaseRegistry[source]¶

Bases: BaseRegistry

The DictBaseRegistry provides an key-value storage for registered objects. These objects will be stored in the order they are being registered.

Furthermore an instance of a DictBaseRegistry provides container features like

some_item = my_dict_registry['some-item']

other_item = my_dict_registry.get('other-item')  # None
other_item = my_dict_registry.get('other-item', 'default')  # 'default'

my_dict_registry['some-item'] = 'something else'

del my_dict_registry['some-item']

as well as

if 'some-item' in my_dict_registry:
    # Do something

for key in my_dict_registry:
    val = my_dict_registry[k]
